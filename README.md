# AI infrastructure capital allocation framework

## Overview

This repository studies how capital should be allocated across the AI infrastructure value chain when power availability becomes the main constraint on growth.

It is not a stock recommendation project. The focus is on physical bottlenecks that matter for investment research: power availability, grid capacity, cooling demand, interconnection timelines, and other limits that can slow data center expansion.

The goal is to build a process that can be reused across infrastructure categories. Each module starts with a thesis, checks it against primary sources where possible, and turns the evidence into allocation questions.

## Research question

How should long-term capital allocate across the AI infrastructure value chain when power availability becomes the primary scaling constraint?

## Research process

Each module follows the same process:

1. Define the investment thesis.
2. Collect evidence from primary and authoritative sources.
3. Evaluate source reliability and evidence type.
4. Identify supporting evidence and key uncertainties.
5. Build a repeatable decision framework.
6. Translate the research into capital allocation implications.
